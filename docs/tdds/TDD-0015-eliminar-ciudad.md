---autor: Equipo SupplyCyclefecha: 2026-05-22titulo: Eliminar Ciudad---# TDD-0015: Eliminar Ciudad## Contexto de Negocio (PRD)### ObjetivoEliminar una ciudad que ya no está en uso.### User Persona- **Nombre**: Administrador- **Necesidad**: Quitar una ciudad mal creada o que ya no corresponde.### Criterios de Aceptación- Debe rechazar si hay domicilios asociados a la ciudad- Si no tiene dependencias, se elimina físicamente (hard delete)## Diseño Técnico (RFC)### Contrato de API- **Endpoint**: `DELETE /api/v1/ciudades/:id`- **Auth**: JWT (rol ADMIN)- **Response** `200 OK` con mensaje de éxito- **Response** `409 Conflict` si tiene domicilios asociados:```json{  "error": {    "code": "CONFLICT",    "message": "No se puede eliminar la ciudad porque tiene domicilios asociados"  }}```## Plan de Implementación### Backend1. Service: verificar `prisma.domicilio.count({ where: { ciudadId: id } })` → si > 0, throw CONFLICT2. Si ok, `prisma.ciudad.delete()`3. Tests: eliminación exitosa, ciudad con domicilios### Mobile4. Botón con confirmación y manejo de error CONFLICT
