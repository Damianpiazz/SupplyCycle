---autor: Equipo SupplyCyclefecha: 2026-05-22titulo: Listar Clientes---# TDD-0003: Listar Clientes## Contexto de Negocio (PRD)### ObjetivoPermitir consultar todos los clientes registrados con filtros y paginación para gestionarlos desde el panel de administración y seleccionarlos al crear pedidos.### User Persona- **Nombre**: Administrador / Repartidor- **Necesidad**: Buscar clientes por nombre o teléfono, navegar la lista y seleccionar uno para asignarle un pedido.### Criterios de Aceptación- El sistema debe retornar una lista paginada de clientes activos- Debe permitir filtrar por nombre (búsqueda parcial), apellido, teléfono y estado- Debe soportar paginación con `page` y `pageSize`- No debe incluir clientes soft-deleted## Diseño Técnico (RFC)### Modelo de Datos**Cliente**| Campo | Tipo | Restricciones ||---|---|---|| id | Int | PK, autoincrement || nombre | String | NOT NULL || apellido | String | NOT NULL || telefono | String | UNIQUE, NOT NULL || estado | EstadoCliente | DEFAULT activo || createdAt | DateTime | @default(now()) || updatedAt | DateTime | @updatedAt || deletedAt | DateTime? | |### Contrato de API- **Endpoint**: `GET /api/v1/clientes`- **Auth**: JWT (cualquier rol autenticado)- **Query Params**: `?search=&estado=&page=1&pageSize=20`- **Response** `200 OK`:```json{  "data": [    {      "id": 1,      "nombre": "Juan",      "apellido": "Pérez",      "telefono": "3512345678",      "estado": "activo"    }  ],  "total": 1,  "page": 1,  "pageSize": 20}```### Estructura del Código (Backend)```src/features/clientes/├── clientes.routes.ts        ← GET / (authMiddleware)├── clientes.controller.ts    ← list(): extrae query params, llama service├── clientes.service.ts       ← list(filters): construye where dinámico con Prisma, paginación└── clientes.schema.ts        ← listClientesSchema (Zod: search, estado opcionales)```### Estructura del Código (Mobile)```features/clientes/├── clientes-service.ts       ← list(params): GET /clientes├── hooks/│   └── use-clientes.ts       ← useQuery(['clientes', params], () => list(params))└── screens/    └── lista-clientes.tsx    ← FlatList con búsqueda y filtros```Pantalla `app/(tabs)/clientes/index.tsx` con lista, search bar y filtro por estado.## Casos de Borde y Errores| Escenario | Resultado Esperado | Código HTTP ||---|---|---|| Sin clientes registrados | `{ "data": [], "total": 0 }` | 200 OK || Filtro sin resultados | `{ "data": [], "total": 0 }` | 200 OK || page menor a 1 | `{ "error": { "code": "VALIDATION_ERROR" } }` | 400 Bad Request || pageSize mayor a 100 | Se ajusta a 100 | 200 OK || Sin token JWT | `{ "error": { "code": "UNAUTHORIZED" } }` | 401 Unauthorized |## Plan de Implementación### Backend1.
