---autor: Equipo SupplyCyclefecha: 2026-05-22titulo: Eliminar Domicilio---# TDD-0010: Eliminar Domicilio## Contexto de Negocio (PRD)### ObjetivoPermitir eliminar un domicilio de un cliente. Al ser un dato histórico, se usa soft delete para mantener la trazabilidad.### User Persona- **Nombre**: Administrador- **Necesidad**: Quitar un domicilio que el cliente ya no usa.### Criterios de Aceptación- Soft delete: marcar `deletedAt`- No debe afectar pedidos históricos asociados a ese domicilio## Diseño Técnico (RFC)### Contrato de API- **Endpoint**: `DELETE /api/v1/domicilios/:id`- **Auth**: JWT (rol ADMIN)- **Response** `200 OK`:```json{  "data": { "message": "Domicilio eliminado exitosamente" }}```## Plan de Implementación### Backend1. Agregar campo `deletedAt` al modelo Domicilio (migración Prisma)2. Service: `prisma.domicilio.update({ where: { id }, data: { deletedAt: new Date() } })`3. Ruta `DELETE /:id` con auth ADMIN4. Tests### Mobile5. Botón de eliminar con confirmación6. Mutation, invalidar queries, redirigir a detalle del cliente
