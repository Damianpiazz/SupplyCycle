---autor: Equipo SupplyCyclefecha: 2026-05-22titulo: Actualizar Cliente---# TDD-0004: Actualizar Cliente## Contexto de Negocio (PRD)### ObjetivoPermitir modificar los datos de un cliente existente: nombre, apellido, teléfono y estado.### User Persona- **Nombre**: Administrador- **Necesidad**: Corregir el teléfono de un cliente o actualizar su estado a `moroso` o `inactivo`.### Criterios de Aceptación- El sistema debe actualizar solo los campos enviados (parcial)- El teléfono debe seguir siendo único (excepto el del mismo cliente)- Debe rechazar si el cliente no existe- Debe rechazar si se intenta cambiar el teléfono a uno ya ocupado## Diseño Técnico (RFC)### Modelo de Datos**Cliente**| Campo | Tipo | Restricciones ||---|---|---|| id | Int | PK, autoincrement || nombre | String | NOT NULL || apellido | String | NOT NULL || telefono | String | UNIQUE, NOT NULL || estado | EstadoCliente | DEFAULT activo || createdAt | DateTime | @default(now()) || updatedAt | DateTime | @updatedAt || deletedAt | DateTime? | |### Contrato de API- **Endpoint**: `PATCH /api/v1/clientes/:id`- **Auth**: JWT (rol ADMIN)- **Request Body** (todos opcionales):```json{  "nombre": "Juan Carlos",  "telefono": "3519876543",  "estado": "moroso"}```- **Response** `200 OK`:```json{  "data": {    "id": 1,    "nombre": "Juan Carlos",    "apellido": "Pérez",    "telefono": "3519876543",    "estado": "moroso"  }}```### Estructura del Código (Backend)```src/features/clientes/├── clientes.routes.ts        ← PATCH /:id (authMiddleware, authorize('ADMIN'))├── clientes.controller.ts    ← update(): schema.parse(body + params), service.update()├── clientes.service.ts       ← update(id, dto): verifica existencia, teléfono único si cambia, actualiza└── clientes.schema.ts        ← updateClienteSchema (Zod: todos opcionales)```### Estructura del Código (Mobile)```features/clientes/├── clientes-service.ts       ← update(id, dto): PATCH /clientes/:id├── hooks/│   └── use-update-cliente.ts ← useMutation({ mutationFn: (dto) => update(id, dto) })└── screens/    └── editar-cliente.tsx    ← formulario precargado con datos actuales```Pantalla `app/(tabs)/clientes/[id]/editar.tsx`.## Casos de Borde y Errores| Escenario | Resultado Esperado | Código HTTP ||---|---|---|| Cliente inexistente | `{ "error": { "code": "NOT_FOUND" } }` | 404 Not Found || Teléfono ya usado por otro cliente | `{ "error": { "code": "CONFLICT" } }` | 409 Conflict || Body vacío | `{ "error": { "code": "VALIDATION_ERROR" } }` | 400 Bad Request || Estado inválido | `{ "error": { "code": "VALIDATION_ERROR" } }` | 400 Bad Request || Sin token ADMIN | `{ "error": { "code": "FORBIDDEN" } }` | 403 Forbidden |## Plan de Implementación### Backend1.
