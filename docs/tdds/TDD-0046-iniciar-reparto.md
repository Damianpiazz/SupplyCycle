---autor: Equipo SupplyCyclefecha: 2026-05-22titulo: Iniciar Reparto---# TDD-0046: Iniciar Reparto## Contexto de Negocio (PRD)### ObjetivoMarcar el inicio de un reparto cuando el repartidor comienza la ruta. Los pedidos pasan a `en_ruta`.### User Persona- **Nombre**: Repartidor- **Necesidad**: Iniciar el recorrido del día.### Criterios de Aceptación- Al iniciar el reparto se setea `inicio` con la fecha/hora actual- Todos los pedidos del reparto pasan a estado `en_ruta`- No se puede iniciar un reparto ya iniciado o finalizado### Contrato de API- **Endpoint**: `POST /api/v1/repartos/:id/iniciar`- **Auth**: JWT (rol REPARTIDOR o ADMIN)- **Response** `200 OK`:```json{ "data": { "id": 1, "inicio": "2026-05-23T09:00:00Z", "pedidosActualizados": 5 } }```## Plan de Implementación### Backend1. Service: verificar reparto no iniciado → `inicio = new Date()` → actualizar todos los pedidos a `en_ruta` con `prisma.pedido.updateMany()`2. Tests: inicio exitoso, reparto ya iniciado, sin pedidos### Mobile3. Botón "Iniciar reparto" en el detalle4. Confirmación: "¿Iniciar reparto? Los pedidos pasarán a en_ruta"
