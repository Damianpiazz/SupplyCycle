---autor: Equipo SupplyCyclefecha: 2026-05-22titulo: Quitar Item de Pedido---# TDD-0033: Quitar Item de Pedido## Contexto de Negocio (PRD)### ObjetivoEliminar un producto del pedido.### User Persona- **Nombre**: Administrador- **Necesidad**: Quitar un producto que el cliente ya no quiere.### Criterios de Aceptación- Solo se puede quitar items de pedidos en estado `pendiente`- No se puede quitar el único item del pedido (debe quedar al menos 1)### Contrato de API- **Endpoint**: `DELETE /api/v1/pedidos/:pedidoId/items/:itemId`- **Auth**: JWT (rol ADMIN)- **Response** `409 Conflict` si es el último item:```json{ "error": { "code": "CONFLICT", "message": "El pedido debe tener al menos un item" } }```## Plan de Implementación### Backend1. Service: verificar pedido pendiente → contar items actuales → si es el último, throw CONFLICT → `prisma.itemPedido.delete()`2. Tests### Mobile3. Botón "Quitar" con confirmación
