---autor: Equipo SupplyCyclefecha: 2026-05-22titulo: Asignar Horario a Día de Visita---# TDD-0018: Asignar Horario a Día de Visita## Contexto de Negocio (PRD)### ObjetivoDefinir la ventana horaria en que se realiza la visita un día determinado (ej: lunes de 9 a 12).### User Persona- **Nombre**: Administrador- **Necesidad**: Configurar la franja horaria de entrega para cada día.### Criterios de Aceptación- El horario debe tener inicio y fin- La hora de fin debe ser posterior a la de inicio## Diseño Técnico (RFC)### Modelo de Datos**Horario**| Campo | Tipo | Restricciones ||---|---|---|| id | Int | PK, autoincrement || inicio | DateTime | NOT NULL (solo hora) || fin | DateTime | NOT NULL (solo hora) || diaId | Int | FK -> Dia, NOT NULL |### Contrato de API- **Endpoint**: `POST /api/v1/dias/:diaId/horarios`- **Auth**: JWT (rol ADMIN)- **Request Body**: `{ "inicio": "09:00", "fin": "12:00" }`- **Response** `201 Created`## Plan de Implementación### Backend1. Schema Zod con validación inicio < fin2. Service: verificar día → `prisma.horario.create()`3. Tests### Mobile4. Formulario con TimePicker
