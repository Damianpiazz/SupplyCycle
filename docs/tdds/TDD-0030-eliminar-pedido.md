---autor: Equipo SupplyCyclefecha: 2026-05-22titulo: Eliminar Pedido (Soft Delete)---# TDD-0030: Eliminar Pedido (Soft Delete)## Contexto de Negocio (PRD)### ObjetivoEliminar lógicamente un pedido. Solo se permite eliminar pedidos en estado `pendiente` o `cancelado`.### User Persona- **Nombre**: Administrador- **Necesidad**: Limpiar pedidos mal creados o duplicados.### Criterios de Aceptación- Soft delete: marcar `deletedAt`- No se puede eliminar un pedido `en_ruta` o `entregado`### Contrato de API- **Endpoint**: `DELETE /api/v1/pedidos/:id`- **Auth**: JWT (rol ADMIN)- **Response** `409 Conflict` si el estado no lo permite:```json{ "error": { "code": "CONFLICT", "message": "No se puede eliminar un pedido en reparto" } }```## Plan de Implementación### Backend1. Service: verificar estado (solo pendiente o cancelado) → `prisma.pedido.update({ data: { deletedAt: new Date() } })`2. Tests### Mobile3. Botón "Eliminar" solo visible en estados permitidos
